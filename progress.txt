## Codebase Patterns
- Keep new workspace TypeScript configs self-contained; this repo does not currently install `bun-types`, so package-level typecheck should not assume those globals exist.
- When a workspace starts importing Node built-ins directly, add `@types/node` and keep any Bun-only globals limited to the specific shims still needed, such as `import.meta.main`.
- Adaptive-agent delegation logs do not always carry `runId`; normalize a canonical subject run as `runId ?? parentRunId ?? childRunId` before grouping events into run timelines.
- In `packages/analysis`, add new output formats by extending the canonical `buildAnalysisReport` shape first, then render terminal or JSON views from that shared report model.
- In `packages/analysis`, treat `tool.completed` and `run.completed` as potentially failed unless their structured payloads say success; code-executor-style tools often report errors via `output.success = false` instead of emitting `*.failed`.
- Run drill-down mode should resolve selection from the reconstructed run graph, not raw events, so `--run` and `--root-run` can reuse the same child-run tree and failure summaries.
- In `packages/analysis`, build exporter-specific outputs from a shared enriched analysis bundle so Markdown/HTML/CSV and JSON stay aligned on the same run, cohort, and anomaly data.
- In `packages/analysis`, watch mode should diff discovered file fingerprints (`path + size + mtime`) and then rerun the normal full analysis pipeline instead of trying to maintain incremental parser state.
